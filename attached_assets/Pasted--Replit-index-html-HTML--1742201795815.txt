Ты знаешь весь проект на Replit, который состоит из следующих файлов: index.html – базовая HTML-разметка с метатегами, подключениями через CDN (marked.js, Font Awesome), inline-стилями и контейнерами (#app-content, #home, #guide, #day-selection, #task-selection, #global-loading-overlay и т.д.); css/style.css – полный набор стилей (переменные, базовые, адаптивные, стили загрузчиков, карточек, collapsible blocks и т.д.); js/courseManager.js – модуль с классом CourseManager для загрузки, синхронизации и резервного копирования курсов, получения списка профессий, дней и уроков; js/devMode.js – модуль режима разработчика, перехватывающий fetch/XHR, создающий панель логов и устанавливающий глобальный объект devMode; js/admin.js – административный модуль (adminPanel) с горячими клавишами и UI-контролами; js/app.js – основной модуль, реализующий инициализацию (initApp), переключение между разделами (home, day-selection, task-selection, guide), загрузку контента уроков (с обработкой Markdown через createCollapsibleBlocks) и обработку событий; js/fetchTests.js – скрипт для получения тестовых данных с вебхуков и сохранения их в JSON. Задача: отредактируй текущий HTML-файл, объединив в нём весь функционал проекта так, чтобы итоговый файл был самодостаточным и работал как исходное многомодульное приложение. Требования: (A) объединить inline-стили из index.html и css/style.css в один блок <style> в ; (B) встроить весь JS-код из указанных файлов в один или несколько <script> блоков перед </body>, удалив import/export, чтобы все переменные (courseManager, devMode, adminPanel, app, fetchTests) стали глобальными, соблюдая порядок: courseManager → devMode → admin → app → fetchTests; (C) итоговый файл должен поддерживать загрузку курсов (с резервным копированием), режим разработчика, админ-панель, переключение разделов (home, day-selection, task-selection, guide) и получение тестовых данных, при этом все глобальные функции (selectDay, selectLesson, goBackToTaskSelection, openVocabulary, openAdminPanel и т.д.) должны работать корректно; (D) итоговый HTML должен быть самодостаточным (без локальных подключений, только CDN для внешних библиотек) и сохранять все комментарии и форматирование исходного кода.
